# Professional Formatting & Visual Elements



1. **Zero Emojis:** Strictly prohibit the use of emojis in all contexts. This includes code comments, git commit messages, terminal commands, and user interfaces. Maintain a strictly professional, technical, and clean tone at all times.

2. **Use Professional Icons:** When a visual indicator, button icon, or UI element is requested or logically required, you must use proper web icons (recommended Iconify, Lucide, or Material Symbols) rather than falling back to emojis.

3. **Triggering the Search MCP:** If you do not know the exact SVG code or class name for the necessary icon off the top of your head, you must immediately use the ddg-search MCP. Search for the specific icon library documentation, locate the correct icon snippet, and implement it accurately into the codebase. 

4. **Implementation:** Ensure icons are implemented cleanly according to the project's existing frontend stack (e.g., as inline SVGs, component imports, or web fonts) rather than messy workarounds.
